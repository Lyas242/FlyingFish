# 单机压测资源限制诊断与解决方案

## 🔍 **问题分析**

您的分析完全正确！在同一台机器上运行所有服务进行压测，确实会遇到系统级别的资源竞争：

### 当前架构资源消耗：
- **SpringBoot应用**: 800个Tomcat线程 + 业务线程池
- **MySQL**: 数据库连接和处理线程
- **Redis**: 连接处理线程
- **Kafka**: 生产者/消费者线程
- **Zookeeper**: 服务发现线程
- **JMeter**: 压测线程（500个并发用户）
- **操作系统**: 系统进程线程

**总线程数估算**: 2000+ 线程同时运行！

## 🚨 **系统资源限制检查**

### 1. 检查系统线程限制
```bash
# 查看系统最大线程数限制
ulimit -u

# 查看当前用户进程数
ps -eLf | wc -l

# 查看系统总线程数
cat /proc/sys/kernel/threads-max

# 查看每个进程最大线程数
cat /proc/sys/kernel/pid_max
```

### 2. 检查文件描述符限制
```bash
# 查看当前文件描述符限制
ulimit -n

# 查看系统总文件描述符限制
cat /proc/sys/fs/file-max

# 查看当前已使用的文件描述符
lsof | wc -l
```

### 3. 检查内存使用情况
```bash
# 查看内存使用
free -h

# 查看各进程内存使用
top -o %MEM
```

## 🛠️ **立即优化方案**

### 方案一：调整系统限制（立即执行）

#### 1.1 临时提升限制
```bash
# 提升用户进程限制
ulimit -u 32768

# 提升文件描述符限制
ulimit -n 65536

# 查看修改后的限制
ulimit -a
```

#### 1.2 永久修改限制（需要重启）
编辑 `/etc/security/limits.conf`：
```bash
# 添加以下配置
* soft nproc 32768
* hard nproc 65536
* soft nofile 65536
* hard nofile 65536
```

### 方案二：优化各服务的线程配置

#### 2.1 减少SpringBoot线程数（临时测试）
```yaml
# application.yml - 临时降低线程数进行测试
server:
  tomcat:
    threads:
      max: 200                    # 临时降低到200
      min-spare: 20               # 最小空闲线程数
    max-connections: 2000         # 降低最大连接数
    accept-count: 100             # 降低等待队列
```

#### 2.2 优化数据库连接池
```yaml
# shardingsphere配置 - 降低连接数
hikari:
  minimum-idle: 5               # 临时降低最小连接数
  maximum-pool-size: 20         # 临时降低最大连接数
```

#### 2.3 优化Redis连接池
```yaml
# application.yml - 降低Redis连接数
spring:
  data:
    redis:
      jedis:
        pool:
          max-active: 50          # 临时降低
          max-idle: 20            # 临时降低
```

### 方案三：JMeter压测优化

#### 3.1 降低并发用户数进行测试
```
建议先从100个并发用户开始测试
逐步增加到200、300、400、500
观察系统资源使用情况
```

#### 3.2 JMeter JVM优化
```bash
# 修改JMeter启动参数
export HEAP="-Xms1g -Xmx4g -XX:MaxMetaspaceSize=256m"
export JVM_ARGS="-XX:+UseG1GC -XX:MaxGCPauseMillis=100"
```

## 📊 **实时监控命令**

### 监控系统资源使用
```bash
# 实时监控线程数
watch -n 1 'ps -eLf | wc -l'

# 实时监控文件描述符使用
watch -n 1 'lsof | wc -l'

# 监控各Java进程的线程数
ps -eLf | grep java | wc -l

# 监控CPU和内存使用
htop

# 监控网络连接数
netstat -an | grep ESTABLISHED | wc -l
```

### 监控各服务状态
```bash
# 监控Tomcat线程池状态（通过JMX或Actuator）
curl http://localhost:6086/actuator/metrics/tomcat.threads.busy
curl http://localhost:6086/actuator/metrics/tomcat.threads.current

# 监控数据库连接数
# MySQL: SHOW PROCESSLIST;

# 监控Redis连接数
# redis-cli info clients
```
